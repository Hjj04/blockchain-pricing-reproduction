# 基于Stackelberg博弈的区块链车联网数据交易定价模型 — 项目汇总

## 一、项目简介

本项目复现论文《基于Stackelberg博弈的区块链车联网数据交易定价模型》（高春祺，李雷孝，杜金泽）的核心实验。项目使用真实的 EUA（Edge User Allocation）墨尔本CBD数据集，实现了Stackelberg博弈定价算法（Algorithm 1），并通过4个Jupyter Notebook复现了论文中的图4~图10。

### 4.1 `src/data_loader.py` — 数据加载与预处理（48行）

负责加载EUA数据集、抽样数据所有者、创建消费者对象。是整个数据流的入口。

| 函数 | 签名 | 作用 |
|------|------|------|

| `sample_owners` | `sample_owners(df, n_owners=5, seed=42)` | 从 DataFrame 中随机抽样 `n_owners` 个数据所有者，并为每个所有者生成成本 `c_i ∈ [0.5, 2.0]`。使用 `np.random.RandomState(seed)` 保证可复现。返回增加了 `cost` 列的 DataFrame。 |

**数据流向**：`load_eua_data()` → `sample_owners()` 得到5个所有者 → `create_consumer()` 创建消费者 → 传入 `quality_model.compute_quality()` 计算质量因子。

### 4.2 `src/quality_model.py` — 数据质量评估模型（50行）

实现论文公式(12)~(17)，根据数据所有者与消费者之间的空间距离和时间差异，计算每个所有者的综合数据质量因子 Q_i。

| 函数 | 签名 | 对应公式 | 作用 |
|------|------|----------|------|
| `haversine_distance` | `haversine_distance(lat1, lon1, lat2, lon2, R=6371.0)` | 公式(12)-(15) | 计算两个经纬度坐标之间的地表距离（km）。输入为度，内部转弧度。支持numpy数组向量化计算。R为地球平均半径。 |
| `spatial_quality` | `spatial_quality(position_diff, theta_p)` | 公式(16) | 计算空间质量因子：`Q_pos = 1 - distance / θ_p`。距离为0时质量=1，距离≥θ_p时质量=0。结果clip到[0,1]。 |
| `temporal_quality` | `temporal_quality(timediff, theta_t)` | 公式(17) | 计算时间质量因子：`Q_time = 1 - |timediff - θ_t| / θ_t`。当时间差恰好等于θ_t时质量最高=1，偏离越远质量越低。结果clip到[0,1]。 |
| `compute_quality` | `compute_quality(owners_df, consumer, theta_p=10.0, theta_t=60.0, alpha=0.5)` | 综合 | 对外暴露的主函数。对所有数据所有者依次计算空间距离→空间质量→时间差→时间质量→加权综合：`Q_i = α·Q_pos + (1-α)·Q_time`。返回 shape=(N,) 的数组，值域[0,1]，内含assert检查。 |

**计算流程**：经纬度 → Haversine距离(km) → 空间质量[0,1] → 时间差(min) → 时间质量[0,1] → 加权综合Q_i[0,1]

### 4.3 `src/stackelberg_model.py` — Stackelberg博弈定价模型（122行，核心）

这是整个项目最核心的模块，实现了论文的Algorithm 1——Stackelberg纳什均衡梯度迭代算法。模型中数据消费者（买方）是跟随者，数据所有者（卖方）是领导者。

| 函数 | 签名 | 对应公式 | 作用 |
|------|------|----------|------|
| `buyer_utility` | `buyer_utility(Q, x, p)` | 公式(21/26) | 计算消费者效用：`U_buyer,i = Q_i · ln(x_i + 1) - x_i · p_i`。Q越高、购买量越大效用越高，但价格越高效用越低。 |
| `seller_utility` | `seller_utility(Q, x, p, c)` | 公式(18) | 计算卖家基础效用：`U_seller,i = x_i · (p_i - c_i)`，即销量×利润率。 |
| `seller_utility_with_competition` | `seller_utility_with_competition(Q, x, p, c)` | 公式(30)-(31) | 计算含市场竞争力的卖家效用。先算性价比 `w_i = Q_i/p_i`，再算竞争力权重 `W_i = N·w_i / Σw`，最终 `U = W_i · x_i · (p_i - c_i)`。性价比高的卖家获得更大的市场份额。 |
| `optimal_buy_quantity` | `optimal_buy_quantity(Q, p)` | 公式(29) | 消费者最优购买量的解析解：`x_i* = Q_i/p_i - 1`。这是对消费者效用函数求一阶导令其为零的结果。结果clip到≥0。 |
| `seller_utility_y` | `seller_utility_y(Q, y, c, N)` | 公式(33) | 用辅助变量 `y_i = 1/p_i` 重写的卖家效用函数，便于求导。 |
| `gradient_seller_utility_y` | `gradient_seller_utility_y(Q, y, c, i)` | 公式(34) | 计算第i个卖家效用对y_i的偏导数。令 `S = Σ_{z≠i}(Q_z/Q_i)·y_z`，则 `∂U/∂y_i = -N · [Q_i·c_i·y_i² + (2Q_i·c_i·y_i - Q_i - c_i)·S - 1] / (S + y_i)²`。注意S排除自身、分母含y_i、前面有负号-N。 |
| `run_stackelberg` | `run_stackelberg(Q, c, p_max, x_min, x_max, mu, max_iter, tol)` | Algorithm 1 | **主算法**。初始化价格为 `(c_i+p_max)/2`，每次迭代：①消费者用公式(29)算最优购买量；②卖家用公式(34)的梯度做梯度上升更新 `y_new = y + μ·grad`；③clip y到合法范围 `[1/p_max, 1/c_i]`；④判断收敛。返回包含价格/数量/效用历史记录的字典。 |

**算法核心逻辑**（`run_stackelberg`）：

```
初始化: p_i = (c_i + p_max) / 2, y_i = 1/p_i
循环:
  1. 消费者最优响应: x_i* = Q_i/p_i - 1  (跟随者)
  2. 卖家梯度上升:   y_i += μ · ∂U/∂y_i  (领导者)
  3. 约束投影:       y_i ∈ [1/p_max, 1/c_i]
  4. 价格还原:       p_i = 1/y_i
  5. 收敛判定:       max|Δy| < ε → 停止
```

**返回值结构**：
```python
{
    'price_history':          np.ndarray (T, N),   # 每次迭代的价格
    'quantity_history':       np.ndarray (T, N),   # 每次迭代的购买量
    'buyer_utility_history':  np.ndarray (T, N),   # 每次迭代的消费者效用
    'seller_utility_history': np.ndarray (T, N),   # 每次迭代的卖家效用（含竞争力）
    'total_utility_history':  np.ndarray (T,),     # 每次迭代的系统总效用
    'converged':              bool,                 # 是否收敛
    'iterations':             int                   # 实际迭代次数
}
```

### 4.4 `src/baseline_models.py` — 基线对比模型（60行）

实现两个对比模型，用于论文图10的三模型对比实验。两个函数都复用了 `stackelberg_model` 中的 `buyer_utility`、`seller_utility_with_competition` 和 `optimal_buy_quantity`，不重复实现效用计算。

| 函数 | 签名 | 作用 |
|------|------|------|
| `run_random_pricing` | `run_random_pricing(Q, c, p_max, x_min, x_max, max_iter, seed)` | **随机定价模型**：每次迭代独立随机生成价格 `p_i ∈ [c_i, p_max]`，购买量使用消费者最优响应（公式29：`x* = Q/p - 1`）。由于价格每轮随机，曲线大幅波动，不会收敛。 |
| `run_average_pricing` | `run_average_pricing(Q, c, p_max, x_min, x_max, max_iter, n_bids, seed)` | **平均定价模型**：每次迭代模拟 `n_bids`（默认10）个消费者出价，每个消费者的出价范围为 `[c_i, c_i+Q_i]`（与质量成正比），取均值作为本轮价格。购买量同样使用消费者最优响应（公式29）。价格相对稳定但非最优。 |

**关键设计**：两个基线模型的购买量都使用消费者最优响应 `optimal_buy_quantity(Q, p)` 而非随机值，这保证了消费者行为的经济理性——只有定价策略不同，购买决策一致。这样对比才有意义：Stackelberg模型的优势来自其定价策略的优越性，而非消费者行为的差异。

**返回值结构**：与 `run_stackelberg` 相同格式（包含 `price_history`, `quantity_history`, `total_utility_history`）。
